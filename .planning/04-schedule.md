# 6주 개발 일정

> 기준: 10주차 ~ 15주차 / 12주차 중간 발표 / 15주차 최종 발표

## 주차별 계획

### 10주차 — 환경 설정 & 기반 구조
**목표:** 앱이 빈 화면이라도 Expo Go에서 실행되어야 함

| 작업 | 산출물 | 검증 방법 |
|---|---|---|
| Expo + TypeScript 프로젝트 초기화 | 실행 가능한 기본 앱 | `npx expo start` 실행 확인 |
| react-navigation 설치 및 화면 라우팅 구성 | 홈/기록/리포트 화면 이동 | 탭 전환 동작 확인 |
| Claude API 키 설정 및 연결 테스트 | API 응답 콘솔 출력 | 텍스트 프롬프트 응답 확인 |
| 문서 작성: 비전, 요구사항, WBS, 일정 | `.planning/` 파일 4개 | 파일 존재 및 내용 확인 |

**위험 요소:** Expo 환경 설정 오류 → 대응: Expo Snack(웹 에디터) 임시 활용

---

### 11주차 — 음식 인식 핵심 기능
**목표:** 사진을 찍으면 AI가 음식 이름과 칼로리를 돌려줘야 함

| 작업 | 산출물 | 검증 방법 |
|---|---|---|
| expo-image-picker 연동 | 카메라/갤러리 선택 UI | 이미지 선택 후 미리보기 확인 |
| 이미지 → Base64 변환 + Claude Vision 호출 | AI 응답 JSON | 음식 3종 테스트 |
| 응답 파싱 (음식명, 칼로리, 영양소 추출) | 구조화된 데이터 객체 | 콘솔 출력 확인 |
| 인식 결과 수정 UI | 수정 가능한 결과 화면 | 음식명/양 수정 후 칼로리 재계산 확인 |

**위험 요소:** Claude Vision이 한식을 못 맞출 경우 → 대응: 프롬프트 개선 + 사용자 수정 기능으로 보완

---

### 12주차 — 식단 기록 & 중간 발표
**목표:** 하루 식단을 기록하고 총 칼로리를 볼 수 있어야 함 (중간 발표 데모)

| 작업 | 산출물 | 검증 방법 |
|---|---|---|
| AsyncStorage 연동으로 식단 저장/불러오기 | 앱 재시작 후 기록 유지 | 저장 → 앱 종료 → 재실행 후 확인 |
| 홈 화면: 오늘 식단 목록 + 총 칼로리 | 홈 화면 완성 | 3번의 식사 기록 후 합계 표시 확인 |
| 목표 칼로리 설정 화면 | 설정 화면 | 목표 입력 → Progress Bar 반영 확인 |
| **중간 발표 자료 완성** | `docs/presentation/interim.md` | 발표 리허설 1회 이상 |

---

### 13주차 — 영양소 분석 & 차트
**목표:** 탄·단·지 비율 차트와 주간 리포트 화면이 작동해야 함

| 작업 | 산출물 | 검증 방법 |
|---|---|---|
| victory-native 설치 및 도넛 차트 구현 | 탄·단·지 비율 차트 | 오늘 식단 입력 후 차트 반영 확인 |
| 주간 칼로리 막대 차트 구현 | 주간 리포트 화면 | 7일치 더미 데이터로 차트 확인 |
| 아키텍처 문서 + ADR 3개 작성 | `docs/architecture.md`, `ADR-000*.md` | 파일 내용 확인 |

---

### 14주차 — 품질 개선 & 문서 완성
**목표:** 앱이 안정적으로 동작하고, 발표에 필요한 모든 문서가 완비되어야 함

| 작업 | 산출물 | 검증 방법 |
|---|---|---|
| 에러 처리 (API 실패, 네트워크 없음) | 에러 메시지 UI | 비행기 모드에서 동작 확인 |
| 로딩 상태 UI (스피너, 스켈레톤) | 로딩 화면 | API 호출 중 UI 확인 |
| `docs/setup.md`, `docs/deploy.md`, `docs/testing.md` 작성 | 운영 문서 3개 | 문서대로 따라해서 실행 가능한지 확인 |
| `AGENTS.md`, `README.md`, `BONUS.md` 작성 | 루트 문서 3개 | 파일 내용 확인 |

---

### 15주차 — 최종 발표
**목표:** 완성된 앱 데모 + 모든 질문에 직접 답변 가능

| 작업 | 산출물 | 검증 방법 |
|---|---|---|
| 최종 발표 자료 작성 | `docs/presentation/final.md` | 발표 리허설 2회 이상 |
| Q&A 대비 (setup/deploy/testing/architecture) | 백업 슬라이드 5장 | 예상 질문 10개 답변 연습 |
| 데모 시나리오 최종 점검 | 데모 흐름 문서 | 실제 기기에서 3분 데모 완주 |
