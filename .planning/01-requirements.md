# 요구사항 명세 (MoSCoW)

## 핵심 사용자 시나리오

### 시나리오 1 — 식사 기록
> 점심을 먹고 음식 사진을 찍는다. 앱이 음식을 인식하고 칼로리를 보여준다. 틀린 부분만 수정하고 저장한다.

### 시나리오 2 — 일일 현황 확인
> 저녁 식사 전에 앱을 열어 오늘 먹은 칼로리 합계와 목표 대비 남은 양을 확인한다. 영양소 균형도 간단히 체크한다.

### 시나리오 3 — 주간 리포트 분석
> 일요일에 지난 한 주의 식단 패턴을 확인한다. 어느 요일에 과식했는지, 단백질이 부족했는지를 차트로 파악한다.

---

## Must (반드시 구현)

- [ ] 음식 사진 업로드 (카메라 촬영 또는 갤러리 선택)
- [ ] Claude Vision API로 음식명 자동 인식
- [ ] 음식별 칼로리 자동 계산 및 표시
- [ ] 인식 결과 수정 기능 (음식명, 양 직접 수정)
- [ ] 하루 식단 기록 목록 (아침/점심/저녁/간식)
- [ ] 목표 칼로리 설정 및 일일 진행률 표시

## Should (가능하면 구현)

- [ ] 영양소 분석 결과 표시 (탄수화물 / 단백질 / 지방 g)
- [ ] 탄·단·지 비율 차트 시각화
- [ ] 주간 칼로리 섭취 히스토리 차트
- [ ] 과거 식단 기록 열람

## Could (여유 있으면 구현)

- [ ] 목표 유형 선택 (다이어트 / 벌크업 / 유지)에 따른 맞춤 권장 칼로리
- [ ] 음식 즐겨찾기 (자주 먹는 음식 빠른 추가)
- [ ] 식사별 메모 기능

## Won't (이번 프로젝트에서 제외)

- 의료적 진단 또는 처방 기능
- 실시간 식품 가격/구매 연동
- 소셜 기능 (친구 추가, 공유)
- 운동 칼로리 소모 추적
- 백엔드 서버 / 사용자 계정 시스템

---

## 기술 스택 선택

| 영역 | 선택 | 이유 |
|---|---|---|
| 플랫폼 | React Native + Expo | JS 기반, 빠른 셋업, Expo Go로 즉시 테스트 |
| AI | Claude API (claude-opus-4-7) | Vision 지원, 한국어 음식 인식 우수 |
| 로컬 저장소 | AsyncStorage + SQLite | 서버 없이 기기 내 데이터 관리 |
| 차트 | Victory Native | React Native 친화적 차트 라이브러리 |
| 언어 | TypeScript | 타입 안전성, 코드 자동완성 |
