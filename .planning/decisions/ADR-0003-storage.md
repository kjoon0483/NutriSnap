# ADR-0003: 데이터 저장소로 AsyncStorage 선택 (서버리스)

- **상태:** 결정됨
- **날짜:** 2026-05-19

## 맥락

식단 기록 데이터를 어디에 저장할지 결정해야 한다. 선택지는 AsyncStorage(로컬), Firebase, 자체 백엔드 서버 세 가지였다.

## 결정

**AsyncStorage (로컬 저장소)** 를 선택하고, 백엔드 서버는 구축하지 않는다.

## 이유

- **개발 속도:** 서버 없이 클라이언트만으로 완성 가능 → 6주 일정에 적합
- **복잡성 제거:** 회원가입, 인증, DB 설계 등 부가 작업 불필요
- **개인 정보 보호:** 식단 데이터가 기기 밖으로 나가지 않음
- **오프라인 동작:** 인터넷 없이도 기록 조회 가능

## 트레이드오프

- 기기 분실 시 데이터 복구 불가 → 이번 프로젝트 범위에서는 허용
- 여러 기기 동기화 불가 → Won't 항목으로 명시됨
- 저장 용량 한계 (~6MB) → 텍스트 데이터만 저장하므로 충분

## 구현 방식

```typescript
// 날짜별 키로 식단 데이터 저장
const key = `meals_${date}`;  // "meals_2026-05-19"
await AsyncStorage.setItem(key, JSON.stringify(meals));
```
